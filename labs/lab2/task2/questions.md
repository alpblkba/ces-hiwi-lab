# Questions — Lab 2.2 — Processor-Side Interaction and System Validation

    ## Conceptual questions

    1. What is the purpose of this task?
    2. Which part of the design is hardware?
    3. Which part of the design is software or configuration?
    4. What does HLS generate?
    5. What is the role of Vivado?
    6. What is the role of Vitis?
    7. What files are written by the student?
    8. What files are generated by the tools?
    9. What is the smallest testable version of this task?
    10. What would be a reasonable extension?

    ## Task-specific oral questions

    1. What is a BSP and why does it matter?
2. What happens if software uses an old hardware handoff file?
3. How can you tell whether a failure is in hardware or software?
4. What does processor-side validation prove?
5. What does it not prove?
6. Why should the bitstream, platform, and application be kept consistent?
7. What would you rebuild after modifying the HLS block?

    ## Debugging questions

    1. What was the first error message?
    2. Which log file contains the relevant failure?
    3. Did the failure happen during synthesis, implementation, export, or runtime?
    4. Is the issue caused by code, project configuration, environment setup, or board connection?
    5. Can the problem be reproduced from a clean checkout?

    ## Explanation prompt

    Write a short explanation of the task in 5–8 sentences. The explanation should be understandable to a student who knows basic programming but has not used HLS before.
