# Expected Observations — Lab 1.2 — Seven-Segment Display with HLS

    Students should be able to observe and report:

    - whether the repository structure is clear
    - whether the expected tools are available
    - whether GUI forwarding works, if needed
    - which project files are created or modified
    - which generated artifacts appear after each build step
    - where logs and reports are stored
    - whether the final behavior matches the task objective

    ## Task-specific expected observations

    - student can explain every input-output case
- student understands the top function as a hardware boundary
- student can locate HLS reports
- student can explain what artifact is passed to Vivado
- student can describe how the design could be extended

    ## Acceptable partial progress

    For early lab development, acceptable partial progress includes:

    - documented setup path
    - documented expected workflow
    - created project skeleton
    - identified missing environment assumptions
    - listed unresolved tool or board issues
    - written reflection questions
    - prepared instructor-facing notes

    ## Not acceptable

    The following should not be considered sufficient:

    - undocumented GUI-only progress
    - source files without explanation
    - generated project folders with no reproducible steps
    - unclear tool version
    - no record of failures
    - no explanation of hardware/software roles
